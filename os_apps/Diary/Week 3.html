Refactoring …

This online os can be thought of being composed of two parts. First part is the system, this includes the Home page and Recent Apps Page. The second part is the user applications, including everything else.

The project structure should reflect that as much as possible.

This is how it looks like:

src/actions src/reducers

src/system /components /containers

src/applications /components /containers

Actions and reducers folders setup redux state for the whole os.

Components and containers is a pattern used in React. Components hold files responsible for displaying data given to them, and they have no state (or very minimal state). Containers on the other hand, prepare state and other data and pass them down to components to be displayed.

Containers therefore contain most of the logic, whereas components contain most of the styling for an application.

This structure is nice and increases readability and predictability.

Animations

CSS allows for keyframe animations to be defined. These are rather simple, but they do add to the overall experience. Most of the animations are will be fine tuned as I see fit.

The setup includes a file called Animations.css which is available for all system and user applications.

This allows for animations to be reused between different applications.

Hammerjs

Hammer js is a library that helps implementing touch gestures. I want to use this so that cards in the recent apps page can be swiped away to close the application. I'm still figuring out how it works.
