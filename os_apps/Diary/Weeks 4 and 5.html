Weeks 4 and 5 (Feb 18, March 01)

For this period the goal was to improve the notes and calculator apps so that they can be a bit more visually pleasing and have some useful functionality.

Notes app now persists notes in the database (and therefore can allow multiple notes). They are loaded into memory (redux state) when the app is loaded.

Calculator is a slightly modified version of the open source calculator example provided by Andrew Farmer.

Most of the work on these was on thinking what the simple apps should be able to do, and getting the UI elements just right (mostly).

Other things

During this period other issues presented themselves.

The back button for instance, currently just uses the browser history to go back to the previous URL. It does not relate to the current app that is running. This is not how operating systems should work.

Therefore this is something that needs to be solved somehow.

The app that's running should somehow override the role of the back button to do what is suitable for it.

The cards that show the recent apps also needed improvements. Using react-draggable, it was simple to make them “draggable” in the x-axis direction, although some issues remain to be solved when scrolling as that interferes with the react component thinking the user is trying to drag the card in the y-axis.

Animations in react can be controlled in a more precise manner than simple CSS animations using a module that it has. I also should find time for researching this.

HTML iframes if implemented correctly (wrapped in react components) can make the desktop experience very close to that of normal desktop environments.
